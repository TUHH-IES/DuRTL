# 📋 Software Management Plan (SMP)

## 📌 Scope
Defines how the DuRTL project is managed across **GitLab (dev)** and **GitHub (open source)**.

---

## 🧩 Governance
- **Internal development:** GitLab → controlled team access  
- **Public release:** GitHub → open-source release 

---

## 🔐 Access Control
- Members with role 'Maintainer' can merge changes to master 

---

## 🔄 Branch Strategy
- `master`: Stable - release ready
- `feature/*`: New features/Tasks
- `Bug/*`: Issues

---

## 🧪 Build - Test GitLab
- Create merge request from Feature/Issue branch
- Automatic build-test run for changes/update in merge request or push to master
- In addition to the tests run, style checks like FORMAT and LINT included

---

## 🗂 Documentation
- Build - SetUp - Quick Run support is provided by the steps mentioned in `Readme.md`
- Internal functionality documents can be generated by executing `/doc/doxyfile`

---

## 🏗 Change Management
- Issues/Features drive all work
- Merge/Pull requests mandatory( exception only for documentation update)
- No direct pushes to `master`
