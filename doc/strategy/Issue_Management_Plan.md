# 🗂 Issue/Task Management Plan (IMP)

## 🎯 Purpose
Define how bugs, features, and tasks are tracked, prioritized, and resolved.

---

## 📝 Types
- **Issue** 🐞: Bug
- **Task** 🔧: New functionality or feature, Refactor, upgrade, or infra work  

---

## 🏷 Labels
| Label | Meaning |
|-------|---------|
| `Bug/Bug Fix` | Something broken |
| `Feature` | New feature |
| `High Priority` | Needs immediate attention |
| `Low Priority` | Needs update/fix but not immediately |
| `Refactoring` | Code restructuring and improvement |
| `IFT` | Features or Issues related to IFT code |
| `SMT2` | Features or Issues related to IFT SMT code |
| `Simulation` | Features or Issues related to IFT Simulation code |
| `Tooling` | Features or Issues related to sample package library update |

---

## 🔄 Workflow
1. **Open Issue** (title + description + labels + linked items)
2. **Assign** to owner
3. **Link commits** by  creating Merge Request and linking with the corresponding branch where the issue is being addressed
4. **Close Issue** after verification + merge

---

## 🗓 Milestones
- GitLab → internal issue or feature updates
- GitHub → release milestones (vX.Y.Z -> vMAJOR.MINOR.PATCH)  

---

## 💬 Communication
- Discussions inside issues  
- Team integration using weekly meeting for high-priority tasks
